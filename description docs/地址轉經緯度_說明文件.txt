☆★☆ Python「地址轉經緯度」程式 ★☆★
   示範如何透過爬蟲 Google Map 網頁
   批次獲取大量地址的經緯度，
   並分段儲存成csv文件。

※ 注意事項
   (1)在寫入csv文件前，不可持續開啟。
      否則程式會無法寫入我們正在看的csv檔。
      (若不太走運，剛好程式中止，
       程式會接著寫入一大批資料進csv文件。
       但因為我們正在使用，這一批就無法寫入。)

   (2)為避免被封鎖我們的IP，勢必得暫停一段時間再接著爬
      (否則爬沒幾個地址就會被封了)，請務必耐心等待~~

※ 程式暫停
   為了避免短時間發送過多封包給 Google Map，
   我設置了time.sleeep(秒數)。
   當每次獲取 2~5組(隨機)地址的經緯度後，
   便隨機隨機暫停一段時間休息。
   (比較不容易被擋下，也不會給google map造成太大的負擔)

※ 程式停止
   我們可以  (1)自行按下Ctrl+V暫停 
   也可能    (2)被Google Maps網站擋下
		(對方伺服器偵測到我IP流量過大，
	         或請求太過頻繁，
                 暫時無法以當前的IP訪問Google Maps。)
                => 可以換組IP再繼續...
                   簡單用VPN工具切換IP即可。

※ 運作原理
   某個想做的項目需要將多個地址(台灣)轉成經緯度。
   且懶得花時間研究如何套API，甚至花錢買API @@。

   我恰好觀察到： 
     Google Map 網站在輸入地址按下enter後，
     大約過三秒，URL就會產生經緯度資訊。
     於是用selenium套件想辦法把網址抓下來過濾，
     經緯度變優雅地大功告成！

※ 分段保存
   用csv儲存(方便pandas套件取用/做其它運算)。
   若「程式停止」:
	將 起始ID ~ 終止ID(抓不到東西)的前一個ID，
   存成字典(dict)，以csv文件保存。

   ☆稍微要注意的地方是: 
     index_1 // 從哪個串列開始(接續)
     index_2 // 從這個串列的哪個位置開始
     兩個index的求算、程式邏輯務必正確
   
   每次要蒐集資料前，先讀取csv，看看有多少資料。
   因為定義ID從1起算，"ID+1"便是上述「起始ID」了!
   如此便可做到「分段保存」~~
